Concrete types

"Experiment" wrappers

This section describes concrete types for holding entire experiments including their clusters and metadata.

PhyOutput

LaskaCore.PhyOutput — Type
mutable struct PhyOutput{T} <: AbstractExperiment{T}
    clusterids::Vector{Int64}
    clusters::Vector{Cluster{T}}
    trigtimes::Vector{T}
    meta::Dict{SubString{String},SubString{String}}
end

Struct for holding Kilosort output preprocessed in Phy. Should be instantiated using the outer constructor LaskaCore.importphy.

Direct field access is not recommended. Basic interface functions include:

  • LaskaCore.clusterids – Returns all cluster ID:s as a Vector.
  • LaskaCore.getcluster – Returns a specific LaskaCore.Cluster.
  • LaskaCore.clustervector – Returns all LaskaCore.Cluster:s.
  • LaskaCore.getmeta – Returns the spikeGLX meta as a dict or a specific entry.
  • LaskaCore.triggertimes – Returns the trigger event times.
  • LaskaCore.ntrigs – Returns the length of the trigger event time Vector.

RelativeSpikes

LaskaCore.RelativeSpikes — Type
mutable struct RelativeSpikes{T} <: AbstractExperiment{T}
    clusterids::Vector{Int64}
    clusters::Vector{RelativeCluster{T}}
    trigtimes::Vector{T}
    meta::Dict{SubString{String},SubString{String}}
    stimtrain::Dict{String,T}
    specs::Dict{String,T}
end

Similar to LaskaCore.PhyOutput. However, instead of LaskaCore.Clusters, LaskaCore.RelativeClusters are used. In these, spiketimes relative to trigger event(s) instead of absolute spiketimes are used. Additionally contains the fields:

  • stimtrain::Dict{String,T} – Dict in the format name/id => time. time should be relative to trigger event. Specified by the user on creation of struct using LaskaCore.relativespikes.
  • specs::Dict{String,T} – Dict containing the time before/after (back/forward) trigger events that spikes are included; as well as number of trigger events (ntrig).

Single cluster wrappers

Cluster

LaskaCore.Cluster — Type
struct Cluster{T} <: AbstractCluster{T}
    id::Int64
    info::Dict{String,String}
    spiketimes::Vector{T}
end

Struct for holding a single Cluster.

Direct field access is not recommended. Basic interface functions include:

  • LaskaCore.id – Returns the Cluster id.
  • LaskaCore.nspikes – Returns the number of spikes in the cluster (Based off length of the spiketimes field).
  • LaskaCore.info – Returns the info of the Cluster from "cluster_info.tsv" as a Dict.
  • LaskaCore.spiketimes – Returns a Vector containing all spiketimes of the Cluster.

RelativeCluster

LaskaCore.RelativeCluster — Type
struct RelativeCluster{T} <: AbstractCluster{T}
    id::Int64
    info::Dict{String,String}
    spiketimes::Vector{Vector{T}}
end

Struct for holding a cluster and its spiketimes relative to triggers. Similar to Cluster{T} except that the field spiketimes is a Vector{Vector{T}} where each vector represents trigger #n.

Direct field access is not recommended. Basic interface functions include:

  • LaskaCore.id – Returns the Cluster id.
  • LaskaCore.nspikes – Returns the number of spikes in the cluster (Based off length of the spiketimes field).
  • LaskaCore.info – Returns the info of the Cluster from "cluster_info.tsv" as a Dict.
  • LaskaCore.spiketimes – Returns a Vector containing all spiketimes of the Cluster.
