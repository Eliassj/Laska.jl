Concrete types

"Experiment" wrappers

This section descrives concrete types for holding entire experiments including their clusters and metadata.

PhyOutput

Laska.PhyOutput — Type
mutable struct PhyOutput{T} <: AbstractExperiment{T}
    clusterids::Vector{Int64}
    clusters::Vector{Cluster{T}}
    trigtimes::Vector{T}
    meta::Dict{SubString{String},SubString{String}}
end

Struct for holding Kilosort output preprocessed in Phy. Should be instantiated using the outer constructor Laska.importphy.

Direct field access is not recommended. Basic interface functions include:

  • Laska.clusterids – Returns all cluster ID:s as a Vector.
  • Laska.getcluster – Returns a specific Laska.Cluster.
  • Laska.clustervector – Returns all Laska.Cluster:s.
  • Laska.getmeta – Returns the spikeGLX meta as a dict or a specific entry.
  • Laska.triggertimes – Returns the trigger event times.
  • Laska.ntrigs – Returns the length of the trigger event time Vector.

RelativeSpikes

Laska.RelativeSpikes — Type
mutable struct RelativeSpikes{T} <: AbstractExperiment{T}
    clusterids::Vector{Int64}
    clusters::Vector{RelativeCluster{T}}
    trigtimes::Vector{T}
    meta::Dict{SubString{String},SubString{String}}
    stimtrain::Dict{String,T}
    specs::Dict{String,T}
end

Similar to Laska.PhyOutput. However, instead of Laska.Clusters, Laska.RelativeClusters are used. In these, spiketimes relative to trigger event(s) instead of absolute spiketimes are used. Additionally contains the fields:

  • stimtrain::Dict{String,T} – Dict in the format name/id => time. time should be relative to trigger event. Specified by the user on creation of struct using Laska.relativespikes.
  • specs::Dict{String,T} – Dict containing the time before/after (back/forward) trigger events that spikes are included; as well as number of trigger events (ntrig).

Single cluster wrappers

Cluster

Laska.Cluster — Type
struct Cluster{T} <: AbstractCluster{T}
    id::Int64
    info::Dict{String,String}
    spiketimes::Vector{T}
end

Struct for holding a single Cluster.

Direct field access is not recommended. Basic interface functions include:

  • Laska.id – Returns the Cluster id.
  • Laska.nspikes – Returns the number of spikes in the cluster (Based off length of the spiketimes field).
  • Laska.info – Returns the info of the Cluster from "cluster_info.tsv" as a Dict.
  • Laska.spiketimes – Returns a Vector containing all spiketimes of the Cluster.

RelativeCluster

Laska.RelativeCluster — Type
struct RelativeCluster{T} <: AbstractCluster{T}
    id::Int64
    info::Dict{String,String}
    spiketimes::Vector{Vector{T}}
end

Struct for holding a cluster and its spiketimes relative to triggers. Similar to Cluster{T} except that the field spiketimes is a Vector{Vector{T}} where each vector represents trigger #n.

Direct field access is not recommended. Basic interface functions include:

  • Laska.id – Returns the Cluster id.
  • Laska.nspikes – Returns the number of spikes in the cluster (Based off length of the spiketimes field).
  • Laska.info – Returns the info of the Cluster from "cluster_info.tsv" as a Dict.
  • Laska.spiketimes – Returns a Vector containing all spiketimes of the Cluster.
